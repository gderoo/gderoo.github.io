Twitter movie analytics

December 2016

This is an overview of a group project done for the Berkeley class on Data Storage and Retrieval. The idea was to mine data from Twitter to better understand the buzz for a movie prior to its release which can then be used for targeted marketing, merchandising and selecting release dates.

We built a prototype sourcing data from Twitter, online movie databases (IMDB), conducted basic sentiment analysis on the tweets to assess buzz on hollywood productions, and aggregated the basic analysis into a dashboard.

The graph below gives an overview of the architecture:


The main features of the pipeline are the following:

  • Extraction layer: We built python programs to ingest data from IMDB, and Twitter's streaming api, leveraging packages such as tweepy and imdbpy.
  • Storage layer: We tried different databases learnt in the class. Data extracted from IMDB movie database is persisted to postgres layer. For streaming data, we parse the live json data stream, extract key features, analyze the data and store the results back into a postgres database.
  • Analytics layer: We extract the features of interest from the tweet json (e.g. text, location, etc.), clean the data (e.g. special characters, lemmatization and stemming...), and perform a sentiment analysis.
  • Serving layer: We used Tableau Server, and created visualizations avoiding joins in order to improve performance.
