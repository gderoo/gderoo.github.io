Finding Lane Lines (Part 1)

March 2017

This is an overview of the first project of the Udacity nanodegreee. In this project, we use a series of tools to identify lane lines on the road.
Once the pipeline is fully set up, the result looks like this:


There are 5 main steps of the image pipeline:

  • image is converted to grayscale
  • edges are calculated with the Canny edge method
  • a trapezoid mask is applied to focus the lane search
  • lines are found using the Hough Space
  • finally, we "average" the lines found using a regression
We illustrate main steps below

If the line turns too much and lines cross the middle, the current solution does not work. Possible improvements would be to:

  • limit the angles of the Hough Lines
  • do the regression at the end of the canny edge step directly
  • use the continuity of directions to stabilize the algorithm
    • for each image, we calculate the lines, together with a confidence probability (longer lines result in higher confidence)
    • At time t, the lines are then given by doing a weighted average of the lines of Image t, but also Image t-1 and Image t-2 for instance
This will probably be the object of a future project in the degree
